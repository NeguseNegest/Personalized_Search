# Personalized Search Baseline

A very small baseline for DD2477 Project 2: Personalized Search.

This project gives you something tangible:
- a tiny Flask web app
- Elasticsearch-backed keyword search
- a minimal personalization layer based on historical user queries and clicked domains
- a script that can ingest AOL-style logs into a local interaction store
- a tiny demo corpus so you can run the pipeline before touching the full AOL dataset

## What this baseline is meant to teach

The goal is **understanding the moving parts** before you build the stronger project:

1. **Documents** are indexed into Elasticsearch.
2. **User interactions** are stored separately as logs.
3. A **user profile** is built from those logs.
4. A query first gets **baseline BM25 retrieval** from Elasticsearch.
5. The top hits are then **re-ranked** with a simple personalization bonus.

## Important note about the AOL dataset

The AOL query log is excellent for personalization experiments, but it is **not** a complete searchable web corpus. It mainly gives you user IDs, queries, timestamps, clicked ranks, and clicked URLs/domains. In practice, that means:

- use AOL to build **user/session profiles and click signals**
- use a **separate document corpus** for the actual search index

This baseline follows that approach.

## 1. Create a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 2. Configure environment variables

Copy `.env.example` to `.env` and fill in your values, or point the app at the `.env` generated by Elastic `start-local`.

```bash
cp .env.example .env
```

## 3. Create the Elasticsearch index

```bash
python scripts/create_index.py
```

## 4. Load the tiny demo corpus

```bash
python scripts/load_demo_docs.py
```

## 5. Run the web app

```bash
python -m flask --app app.web run --debug
```

Open:

- http://127.0.0.1:5000

## 6. Optional: import AOL logs

If you have downloaded the AOL dataset and converted or extracted a CSV file with columns
`AnonID,Query,QueryTime,ItemRank,ClickURL`, you can import a small subset into the local interaction store.

```bash
python scripts/import_aol_logs.py --input /path/to/aol.csv --limit 5000
```

This writes normalized interactions to `data/interactions_from_aol.jsonl`.

## How the baseline ranking works

### Stage 1: retrieval
Elasticsearch retrieves top documents using BM25 over `title`, `content`, and `topics`.

### Stage 2: re-ranking
For a selected user, the app builds a simple profile from past interactions:
- frequent query terms
- clicked domains

Each candidate result gets a personalization bonus from:
- overlap between the document text and the user profile terms
- whether the document domain matches domains the user clicked before

Final score:

```text
final_score = elasticsearch_score + term_bonus + domain_bonus
```

This is intentionally basic. It is a teaching baseline, not the final project model.
